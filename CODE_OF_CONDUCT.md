# Code of Conduct

This project follows a simple standard: be direct, respectful, and useful.

## Expected Behavior

- Keep discussions technical and constructive.
- Assume good faith when reviewing bugs or false positives.
- Be specific about evidence, examples, and tradeoffs.
- Respect maintainers' time by keeping issues focused.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Spam or unrelated promotion.
- Publishing private vulnerability details before maintainers have had time to respond.

## Enforcement

Maintainers may edit, hide, lock, or delete comments and issues that violate this code of conduct. Repeated or severe violations may lead to a ban from the project.
